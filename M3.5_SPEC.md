> 定位：**M3.5**（M3 与 M4 之间的性能优化轮）；M4 仍为最终交付（报告 + 截图 + 复现核对 + 打包 `学生1_学生2.zip`），不在本 SPEC。
> 评审放行点：本 SPEC 经用户确认后才进 tasks/实现（沿 spec-driven 惯例）。

## 1. Objective（目标与使用者）

一句话：在不动判别逻辑、不破向后兼容的前提下，把两个 guard skill 在**可复现性 / 触发准确性 / 输出稳定性 / 运行效率**四个性能轴上各推进一格，每项都有可量化的 before→after。

- 使用者：① 调用 guard 的 agent（受益于更准的触发）；② 复核的评分者/队友（受益于可复现的 run_metadata 与一致的 trigger 文档）；③ 跑全量 judge 的乙自己（受益于并发提速）。
- 本轮**四项工作**（用户 2026-06-16 选定）：W1 可复现性硬化 · W2 触发优化 · W3 量化/GPU 方差 smoke · W4 judge 并发（**已授权，取代 spec R7「不引入并发」**）。**不含** SKILL.md 上下文瘦身（用户排除）。

## 3. 工作项契约

### W1 — 可复现性硬化（Core）
- **缺口**：`run_metadata.env.model_revision = null`（实测，2026-06-15 各 run_metadata）；换镜像/版本无法复现。
- **做什么**：载入时解析并回写 ① `model_revision`（HF 解析到的 commit hash，经 `huggingface_hub` 或 `from_pretrained` 的 `config._commit_hash`）；② `transformers`/`bitsandbytes`/`accelerate` 版本；③ 可选 `weight_sha256`（**config.json + 首分片**哈希，不哈全 8GB——见 §9-1）。
- **落点**：两侧 `scripts/guards/{llama_guard,shieldgemma2}.py` 暴露 revision；`scripts/main.py` 的 env 块写入。`rule`/`caption-rule`（无权重）该字段为 null（如实）。
- **验收**：真跑后 `env.model_revision` 非 null 且稳定；离线测试不依赖网络（mock）。

### W2 — 触发优化（Plus: description + Core: 文档）
- **缺口**：盲测 7/8，P8「Compute AUROC for the moderation model's predictions」未触发（"不够直接"）；且 `trigger-eval.md` 第 3 行仍写"未实测"，§5 已有 6/15 结果——文档不一致。
- **做什么**：① 调两侧 SKILL.md 的 `description` frontmatter，覆盖**"评估已有预测 / 算 AUROC·FPR / 给 guard 输出打分 / 跑 metrics"** 这类指标计算型措辞（目前偏"跑 guard 判别"）；② trigger-eval.md §1 增对应正例、修第 3 行状态与 §5 一致。
- **硬约束**：**不得增加负例误触**（负例 0 误触是既有强项）——description 只扩"评估/指标"语义，不碰"数据集下载/格式检查"的排除边界。
- **验收（可门控部分）**：description 覆盖指标评估用例；trigger-eval.md §1 增正例、第 3 行状态与 §5 一致；**负例边界未动**（自检）。
- **报告项（非通过条件，必改）**：人工新会话盲测**移出 M3.5 通过门**，改为**报告/回归追踪项**——`trigger-eval.md §5` 以 **before/after** 记录（before = 6/15 的 7/8、P8 漏；after = description 调优后复测结果），仅供报告与触发回归对比，不阻塞 M3.5 交付（顺延 M4 前人工执行）。

### W3 — 量化/GPU 方差 smoke（Core, opt-in）
- **缺口**：int8/GPU 输出 run-to-run 漂移**未测过**（确定性是输出稳定性的一部分）。
- **做什么**：opt-in 测试（`SHIELDGEMMA2_LIVE=1` 同款门控）——对**固定 N=5 张图、int8 连跑 K=3 次**（N/K 写死在测试常量，便于复现），记录 ① 每策略 yes-prob 的 run-to-run **极差/标准差**；② **flip_rate**＝`is_unsafe` 二分判定在 K 次间发生翻转的图占比（决策级稳定性，比连续方差更直接）。结论落 `references/shieldgemma2-notes.md`。
- **验收**：离线默认 skip；开门后产出 **方差(极差/std) + flip_rate** 数字（期望方差≈0、**flip_rate=0**=确定性；若非 0 则量化非确定性如实登记，与 §4-2a 量化叙事呼应）。

### W4 — judge 并发（Plus，已授权，取代 R7）
- **缺口**：judge 22.5s/条 → 全量 1959 串行 ≈11h，本轮只能 120 子集标 partial。
- **做什么**：给 `llm_judge` 的批路径加**有界并发**——`main.py` 增 `--judge-concurrency N`（**默认 1 = 现状串行，向后兼容**）；用有界线程池/`concurrent.futures`，每请求保留既有重试与超时，**输出按 id 确定性排序**（并发不得重排/重复/丢行）。
- **限流/鲁棒**：对代理 429/超时按既有 record-level error 规则降级；并发上限保守（见 §9-2）。
- **偏差登记**：**本项取代 M2_SPEC R7「不引入并发，Ask-first」**——用户 2026-06-16 显式授权，登记于本 SPEC §9 与 M2_summary 偏差台账。
- **验收**：① `--judge-concurrency 1` 输出与历史串行**逐 id 一致**；② `N=4` 子集**实测壁钟提速达硬门 ≥30%（目标 ≥2×）**——达不到则视为未交付，回查限流/池化；③ 输出行集合与串行**完全相同**（按 id 规范化，**不重排/不重复/不丢行**）；④ **`counts.success`(predicted)/`errors`/`skipped` 与串行逐项守恒一致**——并发只改壁钟、绝不改判定结果与计数。

## 6. Code Style

沿既有：受控复制文件保持双侧同步（main.py env 块、若 judge 逻辑被两侧共享）；**纯增量、默认值向后兼容**；并发用 stdlib `concurrent.futures.ThreadPoolExecutor`（judge 是 I/O 绑定、urllib，无需第三方）；env-only 凭证不落文件。

## 7. Testing Strategy

- **TDD**：W4 先写"串行等价 + id 一致 + 并发不重排"的 mock-transport 测试再实现；W1 写"revision 出现在 run_metadata"的 mock 测试。
- **答案钥纪律**：W4 若引入任何聚合数字，双路核算锁定。
- **无回归**：现有 **shieldgemma2 94 / llama-guard 90** 套件保持绿。
- W3 opt-in（默认 skip，不进 CI 关键路径）。

## 8. Boundaries

- **Always**：W4 默认 `--judge-concurrency 1`=串行（向后兼容）；W1 纯增 metadata；受控复制双侧同步；leak 卫生（judge 凭证 env-only、交付前 `git grep` 零命中）；现有套件保持绿。
- **Ask first**：把并发默认值提到 >1；任何**负例触发边界**的改动；改 guard 判定逻辑；权重全量 SHA256（慢）。
- **Never**：提交凭证/权重/真数据；并发导致输出重排/重复/丢行；破坏 94/90 套件；description 改动引入负例误触。

## 9. 决议 / 偏差登记

1. **weight SHA256 口径**：默认哈 `config.json` + 首个 safetensors 分片（快、足以辨版本），**不哈全 8GB**；若仍嫌慢则降级为只记 model_revision。
2. **judge 并发上限**：MiMo 代理限流未知 → 默认建议 `N=4`、上限保守，遇 429/超时按 record-level error 降级；不做指数退避以外的激进重试。
3. **取代 R7**：M2_SPEC R7「judge 不引入并发，Ask-first」→ 用户 2026-06-16 **显式授权并发**；本项落地后于 M2_summary 偏差台账标注"R7 superseded by M3.5 §3-W4"。
4. **W2 复测属人工**：description 调优的真实触发率需新会话盲测，列 M4 前人工项（与既有 trigger eval 同轮）。
