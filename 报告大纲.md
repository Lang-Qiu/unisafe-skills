## 详细大纲

### 0. 引言 / 项目概述（~400 字）
**目的**：交代作业目标、统一叙事、模态选择与分工，让读者一句话抓住"平台"是什么。
**内容要点**：
- 作业=两人一组、方向 A（数据集→统一 JSONL）+ 方向 B（Guard→统一安全判断）配套流水线；B 必须在 A 构造的数据集上评测。
- 方案=**文本核心 + 多模态亮点**；统一 schema + 统一 guard-result 接口跨模态泛化。
- 一句话分工：甲=两个 dataset skill；乙=两个 guard skill；约 50/50。
**证据文件**：`团队分工计划.md`、`M0_接口约定.md`、`M1_summary.md`（§ 目标）。
**承接**：引出"先看数据集如何统一"。

### 1. 数据集构建说明与分析（方向 A）（~750 字）
**目的**：说明三个数据集如何统一、为何这样选，并分析其对评测能力的支撑。

> 🅰️ **本节留空，留给甲（方向 A）撰写——乙不代写。** 篇幅 ~750 字；写作可参考下方「证据地图」§1 行的证据文件（`dataset-*/SKILL.md`、`M0_接口约定.md` schema+映射表、规模/基准率/checker PASS/caption caveat）。建议覆盖：选型与统一 schema、规模与基准率（含 safe/探针样本对 FPR/over-refusal 的支撑）、过 `dataset-format-checker` 的质量门。

**承接**：数据就位 → guard 如何在其上判别。

### 2. Guard 方法说明与分析（方向 B）（~900 字）
**目的**：说明各 guard 方法 + 统一接口，重点呈现 **Skill 工程质量**（这是 40% 结构分主战场）。
**内容**：
- 2.1 Guard 矩阵
  - 文本：**Llama Guard 3-1B**（原生 unsafe-token 概率）、**规则基线**（关键词，0/1 无 AUROC，诚实标注）、**LLM-as-judge**（API，自报置信度经方向映射）。
  - 图像：**ShieldGemma 2-4B**（本地 int8，3 策略 dangerous/sexual/violence）+ **caption-rule 零依赖基线**。
- 2.2 统一 guard-result 接口：同一输出 schema + 同一 exit-code 契约（0/2/3）+ 结果状态机（fallback_only→partial→full）。
- 2.3 **Skill 工程质量**（逐条给证据）：
  - 标准 Skill 结构（SKILL.md+scripts/references/schemas/templates/examples/tests）；
  - **受控复制纪律**（validate/metrics/calibrate 两侧镜像 + 答案钥双算）；
  - **双口径指标**（answered_only / failure_as_wrong 防量化失败虚高）（→ **fig2**：失败率越高落差越大，活例）；
  - **优雅降级/鲁棒性**（gated 失败 exit 2 不崩、逐条 try/except 计数、可复制修复提示、soft timeout）；
  - **可复现性硬化**（run_metadata 记 model_revision+库版本，种子，--limit smoke）。
**证据文件**：`guard-llama-guard/SKILL.md`、`guard-shieldgemma2/SKILL.md`、`guard-llama-guard/references/{llama-guard-notes,llm-judge-notes,metrics-definitions,io-contract}.md`、`guard-shieldgemma2/references/{shieldgemma2-notes,io-contract}.md`、`M1_summary.md`、`M3.5_summary.md`（W1 可复现性）。
**承接**：方法与接口讲清 → 进入实验流程看真实表现。

### 3. 实验流程、结果报告与分析（**重心**，~1550 字）
**目的**：按里程碑讲"流程"，用真实数据矩阵讲"结果"，并诚实定位上限。
**内容**：
- 3.1 实验流程（里程碑链，体现工程推进）
  - M1 文本端到端跑通 → M2 多 Guard 对比 + over-refusal + 对抗/类别 → M3 多模态亮点 → **M3.5 性能优化** → **M3.6 判别效果优化**。
  - 强调：先文本扎实、多模态限小样本，别拖垮可复现性。
- 3.2 文本侧结果（真数据全量 1959 + 探针 250）（→ **fig1** guard 质量对比 Acc/F1/Coverage；明细 `result_summary.csv`）
  - **Llama Guard**：AUROC **0.888**、Acc 0.899、Recall 0.660、FPR 0.063；error 仅 2.4%。
  - **over-refusal**（XSTest 250）：6.4%；对抗鲁棒 AUROC **0.919→0.831**。
  - **judge 子集（120）**：召回最高但 FPR 也最高；**对抗 AUROC 0.976→0.633**（判官范式结构性弱点，量化坐实）。
  - ensemble **被实测否定**；**阈值校准是真杠杆**。
- 3.3 图像侧结果（int8 全量 2037 + CPU 对照）（→ **fig3** 失败类型：caption 缺失 + int8 NaN 主导）
  - int8 **AUROC 0.613**（量化下界）vs **CPU bf16 真值 0.719**；**19.4% NaN（395/2037）= 确定性量化伪影**。
  - **三策略天花板**：11 类里 8 类无策略 → AUROC ~0.72 封顶（结构性）。
- 3.4 效果优化 before→after（M3.6 E1，**出厂配置统一战绩表**）
  - llama 阈值 0.5→**0.55**：FPR 0.063→0.056、over-refusal 6.4%→**6.0%**、Macro-F1 ≈持平（precision-favoring，**非 Macro-F1 杠杆**，诚实口径）。
  - sg2 阈值 0.5→**0.30**：Macro-F1 0.480→**0.523**、Recall 0.125→0.198、FPR 守 <0.1。
  - **E3 NaN 回退（M3.7）**：100 NaN 子集实测**恢复 100/100**，子集**覆盖率 0%→100%**、全量投影 **80.6%→~100%**；恢复分 truth-grade（子集 AUROC 0.675）；**覆盖率赢点，非 int8 质量修复**（全量为混精度）（→ **fig4** 文本/图像/图像+E3 覆盖率对照）。
- 3.5 性能与可复现性（M3.5 + REPRODUCTION）
  - judge 并发 `-j4` 实测 **3.86×/74%↓**（守恒+逐 id 一致）；int8 flip_rate=0（确定性）。
  - 全套件 **llama 105 / sg2 100【现 110】绿**；**干净 clone 离线全链复现**（rule 预测→评分）；真数据级复现需 GPU+HF+真数据（如实定位）。
- 3.6 结构上限（**未解决，仅报告定位**）：三策略天花板 / judge 对抗鲁棒 / int8 量化伪影——非校准/工程可解，须换模型/换精度。
**证据文件**：
  - 流程：`M1_summary.md`、`M2_summary.md`、`M3_summary.md`、`M3.5_summary.md`、`M3.6_summary.md`。
  - 文本数：`全量标注结果/wildguardmix_test/out_text_real/metrics/metrics.md`、`.../calibration/calibration.md`、`guard-llama-guard/references/calibration-notes.md`。
  - 图像数：`全量标注结果/unsafebench_test/out_m3_real/metrics/metrics.md`、`.../out_cpu_crosscheck/crosscheck.md`、`guard-shieldgemma2/references/{shieldgemma2-notes,calibration-notes}.md`。
  - 统一战绩 + 诚实口径：`M3.6_summary.md` §2.0、`guard-*/references/best-known-config.md`。
  - 可复现性：`REPRODUCTION.md`。
**承接**：结果与上限讲清 → 收束结论。

### 4. 结论与局限（~300 字）
**目的**：一句话价值 + 诚实局限 + 后续方向，不夸大。
**内容**：统一平台跨模态可用、文本侧强（Llama Guard AUROC 0.888）、工程质量与可复现性扎实；**E3 NaN 回退已落地（M3.7）**——子集实测**恢复 100/100**、覆盖率 **80.6%→~100%（全量投影）**、恢复分 truth-grade（子集 AUROC 0.675 ≈ CPU 真值 0.719），但**这是覆盖率赢点，非 int8 质量修复**（全量为混精度）。局限=三策略天花板（CPU 真值也 ~0.72 封顶）/judge 对抗坍塌（0.976→0.633）/int8 量化漂移，三者属结构性、E3 只补覆盖未解此三者；**后续=换非量化精度（解 int8 漂移）/换多策略模型（解三策略天花板）**。
**证据文件**：`M3.6_summary.md` §5-6、`M3.7_summary.md` §2-3（E3 数字+诚实口径）、各 `best-known-config.md` known_issues。
