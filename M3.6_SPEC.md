## 1. Objective（目标与使用者）

一句话：用已交付的校准工具 `calibrate.py` 把两个 guard 的**默认判定点**从未校准的 0.5 移到实测最优的 max_macro_f1 点（E1），并隔离试验栈升级能否消掉 int8 19.4% NaN（E2）——前者是确定的小幅真实效果提升，后者是高风险大头的探路。

- 使用者：调用 guard 的 agent / 评分者——拿到的默认行为更优（更高 Macro-F1），且仍可 `--threshold` 覆盖。
- **两项工作**：W-E1 默认阈值校准落地（行为变更）；W-E2 栈升级隔离 spike（research，不动主 env）。

## 2. 分层（按风险）

## 3. 工作项契约

### W-E1 — 默认阈值校准落地（Plus）
- **依据 / 选点（实测定，§9-2）**：默认阈值不是 magic number，按 guard 从生产精度 sweep 中选——**llama = max_macro_f1 @ 0.55**（本身生产分）；**sg2(int8) = recall@FPR≤0.1 @ 0.30**（`out_m3_real` int8 分；**不取 max_macro_f1**——它落在 0.05/FPR 38%，安全 guard 不可接受）。**不照搬 CPU bf16 的 0.10**（跨精度误用，§3A）。两个候选点（max_macro_f1 / 高召回）均在 calibration-notes 留档。
- **做什么**：
  1. **Threshold sweep / calibration curve（E1 必做产物）**：两侧各跑 `calibrate.py` sweep，输出每个 threshold 的 **{threshold, Accuracy, Macro-F1, Recall, FPR, coverage/answered_total}** 全表 → 写入 `references/calibration-notes.md` + 运行产物 `out*/calibration/*.{md,csv}`；sg2 用 int8 生产分、llama 用本身生产分（0.55 候选仍须记 sweep 表为证据）。
  2. **改默认判定点**：`DEFAULT_THRESHOLD`（sg2）/ llama adapter 默认 = sweep 选出的生产 `max_macro_f1`；`--threshold` 覆盖保留（`0.5` 复现旧行为）。
  3. **Threshold flip analysis（E1 必做产物，解释性）**：统计默认 `0.5 → 新值` 的翻转——**总翻转数 / safe→unsafe / unsafe→safe / TP recovered / FP introduced / FP removed / FN introduced / borderline 数**；可选 top-k 翻转例**只列 `id, truth, old_pred, new_pred, score, category`（绝不含真实图文内容）**。落 `calibration-notes.md` 的 `Threshold Flip Analysis` 小节 / `reports/calibration/flip_analysis.{md,csv}`。须能**解释 Macro-F1 变化来自哪些翻转**。
  4. **测试翻转处置**：单元测试中"在 0.5 下断言判定"的用例改为**显式传 `threshold=0.5`**（解耦默认、仍测映射逻辑）+ 新增"默认=校准值"专测；黄金 `metrics.sample.json`/`output.sample` 若随默认翻转则双路核算重锁。
  5. **文档**：SKILL.md/io-contract 记新默认 + 理由 + 指向 §3A Calibration Caveats。
- **验收**：① **threshold sweep 表存在**（两侧），默认可追溯到 `max_macro_f1` 点；② **flip analysis 表存在**且能解释 Macro-F1 变化来源；③ 新默认 vs 0.5 的 **Accuracy/Macro-F1/Recall/FPR before→after 对照表**（llama 0.779→0.793、sg2 实测）；④ `--threshold 0.5` 复现旧行为；⑤ **calibration caveats 已写入文档**（§3A）；⑥ 套件全绿（翻转用例显式化 + 默认断言）+ 黄金重锁。

### W-E2 — 栈升级隔离 spike（Spike，限时）
- **做什么**：另建 conda env（如 `E:\LQiu\conda_envs\pytorch_dl_26`）装 **torch≥2.6 + transformers≥4.53 + 匹配 bnb**；**主 env、主 requirements、两 skill 代码零改动**。
- **固定 probe set（可复现，必做）**：**N=20** 固定集——合成 safe fixtures 若干 + 真实 UnsafeBench 小样本（若可用，覆盖 **safe / violence / sexual / general_harm**）+ **优先包含历史 int8 NaN 样本**；若真图不可用则用合成图 + 已有 M3 smoke 样本并**明确标注 limitation**。**记录 probe set 的 id 列表或生成方式**（真图绝不入库）。
- **三向对照（必做）**：同一 probe set 同时在 ① **旧主 env int8**、② **隔离新 env int8**、③ **CPU bf16 reference** 上评估。
- **测 alignment，不只测 NaN count（必做指标）**：每路至少记录 **NaN count / NaN rate；与 CPU bf16 的 label agreement；mean / max 绝对 confidence drift；（有真值则）AUROC 或子集 AUROC；coverage；（可选）latency**；并判断**新 env int8 是否逼近 CPU bf16 reference**。
- **限时**：装环境 + 一轮复测为界；不陷入版本调试泥潭。
- **验收**：① spike env 与主 env **隔离**（主 env 套件仍 **llama 100 / sg2 99** 绿、零污染）；② 结论**三选一 + 数字**落 `shieldgemma2-notes.md`：**`fixed`**（NaN 显著下降且 alignment 改善）/ **`not_fixed`**（NaN 或 drift 仍明显）/ **`blocked`**（环境无法完成）；③ **NaN rate、label agreement、confidence drift、coverage 至少四项数字**落库；④ **不采纳进主线**——升主 env 属 M3.6 边界外、需另 Ask-first。

## 3A. Calibration Caveats（E1 引用数字前必读）

- 阈值是在**当前真数据 / 当前 split / 当前生产精度**上调得；
- 数据分布、safe/unsafe prior、类别比例变化时，**最优阈值可能漂移**——部署到新分布须重新校准；
- `max_macro_f1` 是**均衡**指标选择，**不一定适合所有部署场景**（高风险或偏好高召回的部署可能要别的点）；
- **高召回阈值（recall@FPR≤budget）作为备选档记录，但不设为默认**；
- **CPU bf16 与 int8 分数分布不同，阈值不可跨精度复用**（sg2 必用 int8 生产分校准）；
- E1 是 **post-hoc threshold calibration，不改变模型本身**——结构性上限（ShieldGemma 2 三策略天花板、judge 对抗鲁棒）**不受 E1 影响，本轮仍只能报告定位、不得声称解决**。

## 6. Code Style

## 7. Testing Strategy

- **答案钥纪律**：E1 新默认下的对照数字双路核算；黄金 sample 翻转则重算锁定。
- **解耦默认**：映射/判定逻辑测试改为显式传 threshold（不依赖默认值），另加"默认=校准值"专测——避免默认变更连带误改逻辑测试语义。
- **无回归**：主 env 套件保持 **llama 100 / sg2 99** 绿。
- **E2**：隔离 env 不进套件；结论是文档证据。

## 8. Boundaries

- **Always**：E1 保留 `--threshold` 覆盖（向后兼容，`0.5` 复现旧行为）；新默认在 SKILL/io-contract/notes 留理由 + 漂移注 + 指向 §3A caveats；黄金重锁；主 env 套件绿；calibration-notes/best-known-config 只放聚合；leak 卫生。
- **Ask first**：**升级主 env / 改主 requirements**（E2 采纳）；改 guard 判定逻辑（阈值以外）；改负例触发边界。
- **Never**：提交权重/凭证/真数据；**逐例 flip 行（含真数据 id/label）入库**（落 gitignored `reports/`/`out*/`，committed 只放聚合计数）；E2 spike 污染主 env；默认变更不留文档/不重锁黄金；声称效果提升却不给新默认 vs 0.5 实测对照；**声称结构上限（三策略天花板 / judge 对抗鲁棒）被解决**。

## 9. 决议 / 偏差登记

1. **shieldgemma2 阈值按 int8 生产精度重算**，不照搬 CPU 的 0.10（跨精度误用）；若 E2 修复 NaN 并采纳新栈，则阈值随之以新精度重校准（顺延）。
2. **E1 阈值口径 = 按 guard 分别定（用户 2026-06-16 看 sweep 实测数字后决）**：
   - **llama = `max_macro_f1` @ 0.55**（温和：FPR 0.075→0.056、Macro-F1 0.779→0.793，无争议）；
   - **sg2(int8) = `recall@FPR≤0.1` @ 0.30**（**不取 max_macro_f1**）——int8 sweep 实测 max_macro_f1 落在 thr=0.05 / **FPR 0.383**（38% 良性误报，安全 guard 不可接受）；改取 FPR 受控点：Recall 0.125→0.198、FPR 0.051→0.089、Macro-F1 0.480→0.523。
   - max_macro_f1 与高召回点均作为 calibration-notes 的备选档记录。理由=安全 guard FP 可控优先（§3A）。
3. **E2=隔离 spike，结论先行**；采纳新栈升主 env 属 M3.6 边界外、需另 Ask-first。
4. **E3（NaN 回退）排除**（用户）。
