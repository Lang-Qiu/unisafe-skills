# 团队分工计划 — Skill-Native 安全实验平台期末作业

> 作业：两人一组，方向 A（Dataset Skill）+ 方向 B（Guard Skill）配套完成。
> 评分：Skill 结构与功能完整性 40% · 可复现性 30% · 结果与报告 30%。

## 0. 总原则

- **甲生产、乙消费**；`dataset-format-checker`（exit 0）是两人之间的**硬契约**。
- 共 **4 个 Skill + 1 份报告**，工作量按 ~50/50 切。
- **先定接口，再并行**；用"手写小样本"解耦依赖，谁都不用干等对方（见 `M0_接口约定.md` 与 `M0_dataset_unified_sample.jsonl`）。
- 全篇统一叙事：**同一套 unified schema + 同一个 Guard 结果接口，跨文本与图像两种模态泛化**——直接回应作业"统一实验平台：可发现、调用、组合、评测"的目标。

## 1. 角色与归属

| 成员 | 方向 | 负责的 Skill |
|---|---|---|
| **同学甲** | A · Dataset | `dataset-wildguardmix`（+ XSTest 过度拒答探针）、`dataset-unsafebench`（多模态亮点） |
| **同学乙** | B · Guard | `guard-llama-guard`（+ 规则基线 + OpenAI Moderation）、`guard-shieldgemma2`（多模态亮点） |

文本核心是评分主干（先做扎实），多模态是加分亮点（小而精，证明跨模态泛化）。

## 2. ⭐ 第一里程碑 M0：共同敲定"接口契约"（最关键，别跳过）

2 人项目最常翻车在"各做各的、最后拼不起来"。第一天两人一起定死，产出物已生成在项目根目录：

1. **统一记录 schema**：由 `dataset-format-checker/assets/unified_schema_v1.json` 固定；两人各跑一遍 `--emit-guide` 当共同规范。
2. **B 到底读哪些字段**：见 `M0_接口约定.md` §3 的「评测任务 ↔ 真值字段」对齐表。
3. **22 类 canonical taxonomy 共享映射表**：甲（`subcategory`→22 类）与乙（Guard 标签→22 类）映射进同一张表，不可映射统一记 `other`，原值保留。见 `M0_接口约定.md` §4。
4. **甲先手写样本**：`M0_dataset_unified_sample.jsonl`（已含 text + image 共 7 条，覆盖各标签组合），**乙立刻能开发/测试 Guard，不必等全量数据**。Guard 输出目标见 `M0_guard_output_sample.jsonl`。

## 3. 同学甲（方向 A）任务清单 — 对齐作业 A 的 5 项要求

| 作业要求 | 具体任务 | 复用示例 skill 的模式 |
|---|---|---|
| ① 数据获取 | `load_dataset` 拉 WildGuardMix(gated)/UnsafeBench；支持 `--cache-dir`、HF token、`HF_ENDPOINT` 镜像 | `load_hf_dataset` + `explain_load_error`（gated 修复提示）+ exit 2 |
| ② 数据解析 | 读 `prompt/response/*_label/adversarial/subcategory`；**null 标签行显式跳过并计数**；编码 `errors="ignore"` | 逐条 try/except 跳过、`first_text`、`json_default` |
| ③ 标签映射 | `subcategory`→22 类（`config/category_mapping.json`）；原值存 `raw_label` | `to_unified_format.py` 的映射结构 |
| ④ 格式转换 | 直接输出 **unified 嵌套 JSONL**；prompt-only→`prompt_only_safety`，带 response→`prompt_response_safety` 填满 4 个子标签；**多模态保留 `content.images` 路径 + caption/OCR** | `flat_to_unified` + `_content_hash` + stable id |
| ⑤ 统计报告 | `metadata.json`：样本数、label 比例、按类别/对抗分布、skipped、load_errors | `summarize()` + metadata 写法 |
| ✅ 验收 | 自跑 `../dataset-format-checker` 得 exit 0；写 `tests/test_basic.py` | self-check 命令 |

## 4. 同学乙（方向 B）任务清单 — 对齐作业 B 的 5 项要求

| 作业要求 | 具体任务 | 复用 / 注意 |
|---|---|---|
| ① 统一输入接口 | 读甲的 unified JSONL，按 `task_type`/`modality` 路由到对应 Guard | 先用 `M0_dataset_unified_sample.jsonl` 开发 |
| ② Guard 调用 | Llama Guard 3-1B（本地）、规则基线（stdlib）、OpenAI Moderation（API）、ShieldGemma 2（图像）；**处理超时/重试/空输出/gated 模型** | 仿 `explain_load_error` 写失败提示；空输出→`error` 字段而非崩溃（见样本第 4 条） |
| ③ 输出解析 | 解析成统一 Guard 结果（`prediction.is_unsafe` + `risk_categories` + `confidence` + `raw_output` + `runtime.latency_ms`） | 按 `M0_接口约定.md` §5 的 Guard 输出 schema |
| ④ 标签映射 | Guard 自带标签 → 22 类；不确定显式记 `other`，与甲共用映射表 | 共享 taxonomy 表 |
| ⑤ 评测兼容 | `metrics.py`：Accuracy、Macro-F1、Recall、**FPR**、**AUROC**（连续分）、**Over-refusal Rate**（XSTest）；**多 Guard × 多任务对比表** | AUROC 需 Llama Guard token 概率 / OpenAI scores；规则基线无 AUROC，诚实标注 |

## 5. 共同任务（两人一起）

- **端到端集成 + 截图**：甲的输出直接喂乙的 Guard，跑通全流程并截图（提交硬性要求"体现流程跑通"）。
- **交叉 review**：乙帮甲核 schema 字段语义，甲帮乙核指标公式（尤其 FPR/AUROC 的"正类=unsafe"别搞反）。
- **小组报告**（控制篇幅，作业明说篇幅不计分）：
  - 数据集构建说明与分析 → 甲
  - Guard 方法说明与分析 → 乙
  - 结果报告与分析 → 乙主笔、甲补数据集侧
  - 成员贡献度 → 共同（见 §7）

## 6. 里程碑时间线（建议顺序）

| 阶段 | 内容 | 负责 |
|---|---|---|
| **M0** | 接口契约 + 手写样本 + 提前在 HF 同意 3 个 gated（WildGuardMix / Llama Guard / ShieldGemma 2） | 两人 |
| **M1** | 文本核心 E2E：WildGuardMix→checker PASS→Llama Guard 出第一版指标 | 甲建数据 → 乙接 |
| **M2** | 多 Guard 对比 + over-refusal + 对抗/按类别分析 | 乙为主 |
| **M3** | 多模态亮点：UnsafeBench + ShieldGemma 2（小样本） | 甲建数据 → 乙接 |
| **M4** | 报告 + 截图 + 复现核对（换台机器重跑）+ 打包 `学生1_学生2.zip` | 两人 |

## 8. 风险与依赖（提前规避）

- **gated 访问**：M0 就去点同意，别等到要跑时被 401 卡住。
- **乙依赖甲的输出** → 用 M0 手写样本解耦，乙全程不空等。
- **多模态显存/下载大** → 严格限小样本、放 M3 最后，别拖垮文本主干。
- **指标定义错** → FPR/AUROC 的"正类=unsafe"两人对齐一次。
